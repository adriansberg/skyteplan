    Algorithm (lifted verbatim from current `+page.svelte` lines 43-115 with one substitution):
    - Build `const allEvents: EventWithShooter[] = []`.
    - For each `shooter` in `shooters`:
      - Create `const processedEvents = new Set<string>()` ‚Äî plain `Set`, NOT `SvelteSet`. Per Research A1 + Pattern 6, the deduplication is identical; `SvelteSet` is a reactivity wrapper with no benefit in a pure helper.
      - For each `event` in `shooter.events`:
        - Compute `eventKey = \`${event.name}-${event.shootingDateTime}-${event.targetNumber}-${event.relayNumber}\``.
        - If `processedEvents.has(eventKey)` ‚Üí skip.
        - If `event.name === 'Felt'`:
          - Filter sibling events with name in `['Minne', 'Felthurtig', 'Stang']` matching `shootingDateTime`, `targetNumber`, `relayNumber`.
          - If matches exist: push `{ ...event, shooter, subEvents: relatedEvents.map((e) => ({ ...e, shooter })) }`; for each related event add its key to `processedEvents`.
          - Else: push `{ ...event, shooter }`.
        - Else (non-Felt event): check for a corresponding `Felt` event with same time/target/relay; if found, do nothing (it'll be handled when the Felt event is processed); else push `{ ...event, shooter }`.
        - `processedEvents.add(eventKey)`.
    - Sort `allEvents` by `parseAsLocalTime(a.shootingDateTime).getTime() - parseAsLocalTime(b.shootingDateTime).getTime()`.
    - Return `allEvents`.

    Conventions: tabs for indentation, single quotes, no trailing commas, no `any`, 100-char width. No TODO comments. No `console.*` calls.
