<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server data → `+layout.svelte` | `data.club` originates from `event.locals.club` set by `hooks.server.ts`; layout trusts it because the hook validated against `clubs` map |
| Error path → `+error.svelte` | When the unknown-subdomain 404 fires, layout data is NOT in scope; `+error.svelte` must not depend on it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | Information Disclosure | `data.club.logoPath` rendered in HTML | accept | `logoPath` is a public static URL (e.g. `/clubs/stordalen.jpg`). No PII or secret. |
| T-06-09 | Denial of Service | `+error.svelte` crashing on missing `data.club` | mitigate | Task 2 uses static `/favicon.ico` and does not reference `data.club`. Verified via grep gate. |
| T-06-10 | Cross-Site Scripting | `data.club.name` interpolated into `alt` attribute | mitigate | `name` comes from the static `clubs.ts` source-controlled map (T-06-01). Svelte auto-escapes attribute interpolation. No user input is reachable here. |
</threat_model>
