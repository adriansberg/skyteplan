## Phase 6 UI Delta

Phase 6 is primarily a server-side change. The **only UI change** is in the top bar (`+layout.svelte`):

- Logo `src` changes from static import `stordalenLogo` → `data.club.logoPath` (server-resolved string)
- Logo `alt` changes from hardcoded `"Skytterinfo"` → `data.club.name` (e.g. "Stordalen Skytterlag")
- Club name text is NOT added to the top bar (current layout shows logo only; name is `alt` text only)

Everything else (spacing, typography, color, nav) is unchanged from existing implementation.

## Typography

Two weights only: 400 (body + label) and 600 (heading + display). Labels at 12px are distinguished by size, not weight.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (font-normal) | 1.5 |
| Label | 12px (text-xs) | 400 (font-normal) | 1.25 |
| Heading | 20px (text-xl) | 600 (font-semibold) | 1.2 |
| Display | 28px (text-2xl) | 600 (font-semibold) | 1.2 |

Source: existing components (`text-xs` in BottomTabBar, `text-xl font-semibold` in +error.svelte, `text-lg font-semibold` in error header). Note: existing BottomTabBar uses `font-medium` — executor should remap to `font-normal` to match this contract.

Phase 6 does NOT introduce new typographic elements.

## Top Bar Interaction Contract

Current behavior (Phase 5): static logo, links to `/`.

Phase 6 behavior: identical interaction, only data source changes.

| Element | Before | After |
|---------|--------|-------|
| Logo `src` | `stordalenLogo` (static import from `$lib/assets/stordalen.jpg`) | `data.club.logoPath` (e.g. `/clubs/stordalen.jpg` from `static/clubs/`) |
| Logo `alt` | `"Skytterinfo"` | `data.club.name` (e.g. `"Stordalen Skytterlag"`) |
| Logo dimensions | `h-8 w-auto` | unchanged |
| Logo link target | `/` | unchanged |
| Fallback if logo missing | browser broken-image placeholder | acceptable — no custom fallback needed |

The top bar does NOT gain a visible club name text node. Logo + alt is sufficient.

## Error Page Contract (Unknown Subdomain)

Unknown subdomain `error(404, 'Siden finnes ikke')` reuses the existing `+error.svelte` unchanged.

No new component, no new styling. The error page already renders:
- Logo (currently `stordalenLogo` import — this import in +error.svelte also changes to `data.club?.logoPath` or a static fallback)
- Red bordered error box with Norwegian copy
- "Gå til forsiden" link

**Special case:** When the subdomain is unknown, `data.club` does not exist (the error throws before layout data resolves). The error page logo should fall back gracefully. Contract: use a static `/favicon.png` or omit the logo in +error.svelte — do not attempt to read `data.club`. Planner decides exact approach.
